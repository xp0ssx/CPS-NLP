# CPS-NLP

### Настройка окружения

Выполните следующие команды для создания и настройки окружения и запуска Jupyter Notebook

```bash

# 1) Создание виртуального окружения
python3 -m venv .venv

# 2) Активация виртуального окружения
source .venv/bin/activate

# 3) Установка инструментов для запуска Jupyter
pip install --upgrade pip
pip install notebook

# 4) Установка системных зависимостей для Ollama
sudo apt-get update
sudo apt-get install -y zstd lshw curl

# 5) Установка Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 6) Скачивание модели Qwen2.5:0.5b
ollama pull qwen2.5:0.5b

# 7) Запуск сервера Ollama
ollama serve

# 8) Проверка, что сервер отвечает (в новом терминале)
curl http://127.0.0.1:11434/api/tags

# 9) Регистрация текущего виртуального окружения как ядра Jupyter
python -m ipykernel install --user --name nlp --display-name "Kernel for NLP"

# 10) Запуск Jupyter
jupyter notebook NLP.ipynb
```

После запуска откройте ноутбук и выберите интерпретатор `Kernel for NLP`, если он не выбрался автоматически.

### Инструментарий и среда выполнения
Работа проводилась в Jupyter Notebook на языке Python. Для взаимодействия с языковой моделью использовалась библиотека `requests`.  

### Подготовка окружения
Перед началом работы были установлены системные утилиты `zstd`, `lshw` и `curl` — они требовались для последующей инсталляции Ollama. Сама Ollama была загружена с официального сайта. После установки в локальное хранилище загружена модель `qwen2.5:0.5b`, а сервер Ollama запущен в фоновом режиме. Корректность запуска проверялась отправкой HTTP-запроса к серверу.

### Программная реализация

Для взаимодействия с моделью написана функция `send_prompt`. Её задача — передать текстовый промпт на сервер Ollama и возвратить сгенерированный ответ.  

Пакетная обработка нескольких запросов реализована функцией `run_inference`. Она принимает список промптов, поочерёдно вызывает `send_prompt` и собирает полученные ответы в единую структуру.

### Тестирование и фиксация результатов
С помощью созданных функций было выполнено 10 тестовых запросов. Каждый запрос содержал уникальный промпт; ответы модели фиксировались.  

Все полученные результаты сохранены в файл `inference_report.csv`. Таблица состоит из двух колонок:  
- `prompt` — исходный запрос,  
- `response` — ответ модели.
